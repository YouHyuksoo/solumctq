# 시스템 매뉴얼 리디자인 설계서

## 1. 목적

현재 HelpModal.tsx의 단순 아코디언 구조를 좌측 사이드바 + 중앙 문서형 스크롤 레이아웃으로 전면 리디자인한다.
8개 모니터링 페이지별로 통일된 7개 섹션 양식을 적용하여 가독성과 정보량을 대폭 개선한다.

## 4. 페이지별 통일 섹션 (7개)

모든 모니터링 페이지는 동일한 7개 섹션 구조를 가진다.

### 4.1 개요
- 이 모니터링의 목적과 대상을 1~2문장으로 설명

### 4.2 대상 공정
- 테이블 형태: 공정명 | RAW 테이블명 | PID/BARCODE 컬럼
- 예: FT#1 | IQ_MACHINE_FT1_SMPS_DATA_RAW | PID

### 4.3 판정 기준
- 테이블 형태: 등급 | 조건 | 조치
- 등급 셀에 색상 뱃지 (A=빨강, B=주황, C=보라)

### 4.4 상태 색상
- 색상 원(dot) + 의미 설명 리스트
- 빨강/주황/노랑/초록/보라/회색 등

### 4.5 데이터 흐름
- 데이터 소스 → 필터링 → 판정 과정 설명
- 수리이력(IP_PRODUCT_WORK_QC) 기준 판정 명시 (해당 페이지)

### 4.6 기간/갱신
- 데이터 기간 (당일 08:00 ~ 익일 08:00 등)
- 자동 갱신 주기, 롤링 설정

### 4.7 비고
- 특이사항, 주의사항

## 5. "시스템 개요" 페이지 구성

사이드바 첫 항목. 기존 HelpModal의 개요+등급+사용법을 통합.

### 섹션:
1. **시스템 소개** - CTQ 이상점 모니터링 시스템 목적
2. **등급 체계** - A급/B급/C급 테이블 (등급, 의미, 색상, 조치)
3. **근무 시간** - 08:00 ~ 익일 08:00
4. **사용 방법** - 기존 5단계 가이드
5. **언어 전환** - KO/EN/VI 안내

## 6. 8개 모니터링 페이지 콘텐츠

### 6.1 반복성 (연속불량)
- 개요: FT#1, ATE, IMAGE, SET 공정의 동일위치 연속불량 감지
- 공정: FT(PID), ATE(PID), IMAGE(BARCODE), SETTV(BARCODE)
- 판정: A급 - 동일 Location 연속 NG 2건+ → Line Stop
- 상태: 빨강(A급), 노랑(NG), 초록(정상)
- 데이터흐름: RAW 테이블 NG → IP_PRODUCT_WORK_QC JOIN → LAG 윈도우 함수로 연속 감지
- 비고: 수리이력 등록 데이터 기준 최종 판정

### 6.2 비연속 불량
- 개요: 동일위치 2건+ 이지만 연속은 아닌 경우 감지
- 공정: FT, ATE, IMAGE, SETTV (동일)
- 판정: B급 - 동일 Location 비연속 2건+ → 출하중지, A급 해당 Location 제외
- 상태: 주황(B급), 노랑(NG), 초록(정상)
- 비고: 수리이력 등록 데이터 기준 최종 판정

### 6.3 사고성
- 개요: HIPOT, BURN-IN, ATE 공정의 사고성 불량 감지
- 공정: HIPOT, BURNIN, ATE
- 판정: HIPOT 1건+ → A급, BURNIN/ATE 2건+ → A급, 1건 → B급
- 상태: 빨강(A급), 주황(B급), 노랑(NG), 초록(정상)

### 6.4 원자재 (동일부품)
- 개요: 동일 DEFECT_ITEM_CODE 부품의 반복 불량 감지
- 판정: A급 - 동일 부품 일 3건+ NG, C급 - 90일 누적 3건+
- 비고: 수리이력 등록 데이터 기준 최종 판정

### 6.5 공용부품 (Open/Short)
- 개요: ICT 공정 Open/Short 불량 감지
- 판정: B급 - 동일 불량코드 일 2건+ → 출하중지
- 비고: 수리이력 등록 데이터 기준 최종 판정

### 6.6 지표 (Indicator)
- 개요: 모델별 × 공정별 주간 불량 건수 비교
- 공정: 7개 전체 (ICT, HIPOT, FT, BURNIN, ATE, IMAGE, SET)
- 판정: C급 - 전주 대비 2배 이상

### 6.7 FPY (직행율)
- 개요: 라인별 × 공정별 당일 직행율 모니터링
- 공정: 7개 전체
- 판정: A급 - 직행율 90% 미만 → Line Stop

### 6.8 설비
- 개요: 라인별 공정별 설비 정지시간 모니터링
- 판정: C급 - 일 정지시간 60분 이상

## 7. 기술 구현

### 7.1 컴포넌트 구조
```
HelpModal.tsx (리디자인, ~300줄 이내)
├── 사이드바 렌더링 (페이지 목록)
├── 콘텐츠 영역 (선택된 페이지의 섹션 렌더링)
└── 공통 섹션 렌더러 (NumberedSection 컴포넌트)
```

- 파일 1개로 유지 (HelpModal.tsx)
- 300줄 초과 시 콘텐츠 데이터를 별도 파일로 분리 가능

### 7.2 데이터 구조
i18n 파일의 `navTooltip` 섹션을 확장하여 7개 섹션별 데이터를 구조화.

```typescript
interface ManualPageData {
  overview: string;
  processes?: { name: string; table: string; pidCol: string }[];
  criteria: { grade: string; condition: string; action: string }[];
  statusColors: { color: string; label: string }[];
  dataFlow: string[];
  period: string;
  refreshInterval?: string;
  notes: string[];
}
```

### 7.3 다국어
- ko.ts, en.ts, vi.ts 3개 파일 모두 동일 구조로 업데이트
